Infrastructure — Hosting
GitHub Pages is the ONLY reliable hosting option
Cloudflare Pages was abandoned after persistent 403 errors that could not be diagnosed remotely. GitHub Pages has been 100% reliable throughout. All files are served from profdrtan.github.io/research-methods. Never go back to Cloudflare Pages for this project.
Infrastructure — Cloudflare Workers
Cloudflare Workers is unreliable — avoid as primary data store
The Worker (rm-students.teikkheong.workers.dev) experienced at least two outages during testing and class preparation. ERR_CONNECTION_REFUSED appeared multiple times. The Worker returned 403 from Claude's server environment throughout, making remote debugging impossible. Workers KV had a documented outage on 25-26 June 2026 affecting the India region. Use GitHub API as the primary data store instead. Worker can be used as a secondary/fallback only.
Data Storage — GitHub API
GitHub API is the reliable data store — use scores.json and students.json
Writing JSON files directly to the GitHub repo via the API has been 100% reliable. Every single file push succeeded throughout the entire project. The raw.githubusercontent.com URL serves files publicly with no authentication. The API requires a PAT split across multiple variables to avoid GitHub secret scanning. Pattern: ghRead(filename) reads via raw URL, ghWrite(filename, data) updates via API PUT. Always GET current SHA before PUT.
Quiz System — Architecture
Self-contained quiz with no server dependency is the right design
The final quiz architecture — all 94 questions embedded as JSON in quiz.html, instant right/wrong feedback, section-based home screen, per-section leaderboard — works entirely in the browser with no server needed for the core experience. Scores are POSTed to GitHub after submission. If GitHub is down, students still complete the quiz and see their own score. The Wall of Shame and live question-by-question launch system were abandoned because they required a reliable real-time server which we did not have.
Quiz System — Question Design
94 questions across 6 banks — scenario questions add real value
Original 52 MCQ questions plus 42 new scenario-based questions from the Google Doc. The scenario questions (S01-S42) are significantly more valuable than the original questions because they test application rather than recall. Students cannot guess scenario answers from memorising definitions. The 3 Central Limit Theorem questions added on 26 June filled a genuine gap. Quiz 6 (Mixed Methods) is hidden by default — only reveal if time permits. Each section has 15-19 questions.

Instructor Access — PIN System
Never use type=password for PIN input on Android
The PIN input field on instructor.html used type="password" which caused two problems on Samsung Galaxy Note 5: (1) placeholder dots rendered as typed characters making the field appear already filled, (2) keyboard sometimes failed to appear. The fix is type="tel" with inputmode="numeric". However the PIN system was ultimately abandoned entirely — secret.html has no PIN and relies on URL secrecy instead.
Mobile Compatibility
Samsung Galaxy Note 5 with Chrome — known limitations
Chrome on this device does not support the Listen to This Page feature despite it being documented for newer Chrome versions. Text-to-speech was built directly into answers.html using the Web Speech API (SpeechSynthesisUtterance) as a workaround. The type=password input bug affects all Chromium-based browsers on older Android versions. Hard refresh on Android Chrome is done via three dots menu — not Ctrl+Shift+R. GitHub Pages has a 60-second propagation delay after every push — always wait before testing.
Session Management
Always start a new session with the master prompt
The master prompt at profdrtan.github.io/research-methods/masterprompt.html contains the complete project state. Paste it at the start of every new Claude conversation. Long conversations exhaust context and degrade code quality — this conversation became progressively less reliable as context filled up. Fresh sessions with the master prompt produce better results.
